# 내 AI 서버를 어디서든 똑같이! Docker로 MLOps 첫걸음 떼기
지난 Step 12에서 우리는 FastAPI를 이용해 내 Mac 노트북을 훌륭한 AI API 서버로 만들었습니다. 이제 이 서버를 아마존(AWS)이나 구글(GCP) 같은 클라우드 서버에 올려 24시간 서비스하려고 합니다.

그런데 내 노트북 코드를 클라우드의 Linux 서버로 그대로 복사해서 실행하면 과연 한 번에 잘 돌아갈까요?
"파이썬 버전이 다릅니다", "라이브러리 충돌이 발생했습니다", "OS가 달라서 실행할 수 없습니다." 등 수많은 에러 폭탄을 맞게 될 확률이 99.9%입니다.

**"어? 내 컴퓨터(Mac)에서는 잘 되는데요?"**

개발자들의 이 지긋지긋한 변명을 영원히 없애준 마법의 기술, 그리고 현대 MLOps(머신러닝 운영)와 서버 배포의 절대적인 표준인 Docker(도커)에 대해 알아보고, 우리의 AI 서버를 도커로 포장해 보겠습니다!

## 1. Docker(도커)란 무엇인가?
도커를 가장 쉽게 이해하려면 '밀키트(Meal Kit)'를 떠올리면 됩니다.

김치찌개를 끓이려면 냄비, 가스레인지, 김치, 고기, 양념 등 수많은 준비물이 필요합니다. 주방(OS 환경)이 바뀌면 맛이 달라질 수도 있죠. 하지만 '밀키트'는 요리에 필요한 완벽한 재료와 레시피를 하나의 박스에 포장해 놓은 것입니다. 캠핑장을 가든, 친구 집을 가든 밀키트 박스만 있으면 항상 똑같은 맛의 김치찌개를 만들 수 있습니다.

- **도커 이미지 (Docker Image)**: 우리의 AI 코드, 파이썬 설치 파일, 필요한 라이브러리(FastAPI, PyTorch 등)를 모두 꽉꽉 눌러 담은 '밀키트 박스(설계도)'입니다.
- **도커 컨테이너 (Docker Container)**: 이 이미지를 실행해서 실제로 돌아가고 있는 '격리된 서버 공간'입니다.

Mac이든, Windows든, Linux든 **도커만 설치되어 있다면** 이 이미지를 가져와 똑같은 환경(컨테이너)에서 AI 모델을 실행할 수 있습니다.

## 2. 실습 준비: 밀키트 재료 명세서 쓰기 (`requirements.txt`)
(※ 실습을 위해 Mac에 [Docker Desktop](https://www.docker.com/products/docker-desktop/)이 설치되어 있고 실행 중이어야 합니다.)

도커 이미지를 만들기 위해서는 먼저 우리 모델이 어떤 라이브러리를 쓰는지 명시해야 합니다. Step 12에서 작업했던 폴더에 `requirements.txt` 파일을 새로 만들고 아래 내용을 적어줍니다.
```
# requirements.txt
fastapi
uvicorn
pydantic
transformers
torch
```

## 3. 실전 코딩: Dockerfile (레시피) 작성하기
이제 도커에게 "이 밀키트를 어떻게 조립할 것인가?"를 알려주는 지시서인 `Dockerfile`을 작성해야 합니다. 확장자 없이 정확히 `Dockerfile`이라는 이름으로 파일을 생성합니다.

```cli
# 1. Base Image: 어떤 OS와 파이썬 버전을 바탕으로 시작할 것인가?
# python 3.10 슬림 버전을 가져옵니다.
FROM python:3.10-slim

# 2. 작업 디렉토리 설정: 컨테이너 내부에서 작업할 폴더 위치를 정합니다.
WORKDIR /app

# 3. 라이브러리 설치: requirements.txt를 먼저 복사하고 설치합니다.
# (이유: 코드가 바뀌어도 라이브러리가 그대로면 캐시를 사용해 빌드를 엄청 빠르게 해줍니다.)
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 4. 코드 복사: 우리의 main.py 파일 등 모든 코드를 컨테이너 안으로 복사합니다.
COPY . .

# 5. 포트 개방: FastAPI가 사용할 8000번 포트를 열어줍니다.
EXPOSE 8000

# 6. 실행 명령어: 컨테이너가 켜질 때 마지막으로 실행할 명령어를 적습니다.
# "uvicorn main:app --host 0.0.0.0 --port 8000" 과 같습니다.
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

## 4. 도커 이미지 빌드하고 실행하기!
모든 준비가 끝났습니다. 터미널을 열고 파일들이 있는 폴더에서 도커 명령어를 실행해 봅시다.

#### 1단계: 도커 이미지 빌드 (밀키트 포장하기)
```
# -t 옵션으로 이미지의 이름(my-ai-api)을 지어줍니다. 맨 뒤의 마침표(.)는 현재 폴더를 의미합니다.
docker build -t my-ai-api .
```

이 명령어를 치면 파이썬을 다운받고 라이브러리를 설치하는 과정이 주르륵 올라갑니다.

#### 2단계: 도커 컨테이너 실행 (밀키트 요리하기)
```
# 내 컴퓨터의 8000번 포트(-p 8000:8000)를 컨테이너의 8000번 포트와 연결해서 실행합니다.
docker run -p 8000:8000 my-ai-api
```

터미널에 `Application startup complete.`가 뜨면 성공입니다!
다시 웹 브라우저를 열고 `http://127.0.0.1:8000/docs` 로 들어가 보세요. 내 Mac 환경이 아니라, 도커가 만들어낸 완벽하게 격리된 가상의 Linux 환경 속에서 우리의 AI 서버가 훌륭하게 돌아가고 있습니다.

이제 여러분은 클라우드 서버(EC2 등)를 빌린 뒤, 이 이미지를 업로드하고 `docker run` 명령어 한 줄만 치면 1초 만에 전 세계를 대상으로 AI 서비스를 시작할 수 있습니다!

##  AI 개발자 성장 로드맵을 마치며

정말 고생 많으셨습니다! Step 1부터 Step 13까지, 여러분은 백지상태의 Mac 노트북을 시작으로 다음과 같은 엄청난 기술들을 직접 손으로 구현해 냈습니다.

1. **Phase 1 (데이터)**: 환경을 구축하고 NumPy와 Pandas로 데이터를 만졌습니다.
2. **Phase 2 (머신러닝)**: 경사하강법의 원리를 깨우치고, XGBoost로 생존자를 예측했습니다.
3. **Phase 3 (딥러닝)**: PyTorch의 텐서와 자동 미분을 다루며 인공신경망(MLP)을 구축했습니다.
4. **Phase 4 (최신 AI 트렌드)**: CNN으로 눈을 달고, Transformer로 언어를 가르쳤으며, LLM과 LangChain으로 나만의 RAG 챗봇을 만들었습니다.
5. **Phase 5 (MLOps 배포)**: FastAPI로 서버를 열고, Docker로 어디서든 실행할 수 있게 포장했습니다.

이 로드맵은 실무에서 AI 엔지니어가 겪는 '**데이터 수집 ➔ 모델링 ➔ 서비스 배포**'라는 전체 사이클을 완벽하게 관통하고 있습니다.
